# Gemini 3.1 Pro 审查意见

在认真审阅了 `PROJECT.md`（完整工程约定）、本地代码库现状以及 `gpt5.4-bug-rep.md` 后，经过多轮自我审查验证，我发现 GPT-5.4 的报告在**通信协议层**和**核心状态机**层面抓得很死，但在**“时序数据降采样入库调度”**和**“前端高保真可视化展现”**等同样在 `PROJECT.md` 中以硬性规格写死的设计上，遗漏了若干明显未落地的事项。

以下事项属于 **“项目显然没有依据 PROJECT 写出来，且完全未被 GPT-5.4 发现”** 的遗漏：

## 1. 任务详情页未依规格应用 ECharts `markArea` 标记异常区间
- **现象**：Project 里为了避免运维人员凭空读数据，明确要在图表底底色上做红色高亮提醒。但当前任务详情页的图表中完全没有这层实现，告警触发的历史时间段在图表上无法形成直观的视觉标注。
- **规格依据**：`PROJECT.md` 10.3 任务详情页（时序图表）明确说明：“异常区间标记（ECharts markArea）：在图表上用半透明红色区域标注告警触发时段”。
- **代码脱节事实**：`web/src/views/TaskDetailView.vue` 的 `updateChart()` 生成 ECharts 配置代码中，仅填充了基本的 `latency` 折线和 `packet_loss` 柱状图数据，完全缺少将告警历史转化为 `markArea` 绑入 `series` 的逻辑。

## 2. InfluxDB (agg_1m / agg_1h) 自动降采样任务全无入库调度实现
- **现象**：在设计上，系统理应通过后台服务或 InfluxDB Task 将原始粒度的数据按照 1m/1h 降采样后真正持久化写入聚合桶（保证查询性能和 7d/30d 长周期数据留存）。然而，当前代码只是在查询时做了一个 **临时聚合（on-the-fly fallback）** 策略，降采样入库的任务压根没有被创建。导致那两个设定好的长期数据桶，在运行中变成永远写不进去的空桶。
- **规格依据**：`PROJECT.md` 6.2 明确说到：“InfluxDB Task（自动降采样任务）会自动运行：每 1 分钟：从 raw 聚合写入 agg_1m”。并在 `5.1` 中标注后台有“降采样调度器（触发 InfluxDB Task）”。
- **代码脱节事实**：后端 `server/services/influx_service.py` 内部仅包含查询降频逻辑 `_query_task_data_aggregated_from_raw()`。不管是通过 Flask Backend 的 scheduler 守护进程回写，还是容器启动时利用 `influx CLI` 下发的原生 Continuous Task 查询配置，当前工程均未提供实现，`agg_1m` 纯属摆设。

## 3. 前端图表严重退化：异构协议对应的“多图表组合”缺失
- **现象**：项目约定并不是所有的网络质量都长得一样。由于 HTTP 和 DNS 有更复杂的耗时组成段，规格里针对协议配置了诸如面积堆叠图和状态码散点排布。结果目前代码极度偷懒，强行把 HTTP 和 DNS 的展示全盘降级为了只有一张单纯“延迟/丢包”的 Ping 图表。
- **规格依据**：`PROJECT.md` 10.4 各协议展示的图表 规定：HTTP 必须展示“各阶段堆叠面积图（DNS+TCP+TLS+TTFB）”与“状态码散点图”，DNS 需展示“解析IP变更记录”，ICMP 还要补“抖动折线图”。
- **代码脱节事实**：在 `web/src/views/TaskDetailView.vue` 只有一组固定的宽高占位（`chartRef` 430px），内部逻辑完全不关心当前任务的 `protocol` 是什么，粗暴地把通用的 latency 和 packet_loss 灌给 ECharts，其余专属组合图表无任何 Vue 组件模板或配置分支。

## 4. 全局系统设置（页面标题/副标题）沦为“写进黑洞的表单”
- **现象**：虽然按规格老老实实写完了“系统设置”这套接口并在管理后台能存取 `site_title`。但这套逻辑对前端来说是个瞎子——保存之后，网站真正的菜单栏、标题栏 `document.title` 等根本不生效。管理员改了等于没改。
- **规格依据**：`PROJECT.md` 10.6 关于“页面标题/副标题自定义”的前后联动机制要求。
- **代码脱节事实**：`web/src/views/LayoutView.vue` 的站点顶部直接硬编码写死了 `NSR v{{ appVersion }}`。主系统的生命周期或路由导航（Router 前置补丁）并未尝试加载系统设置并在全局页面布局和原数据头（head）里刷新标题。
