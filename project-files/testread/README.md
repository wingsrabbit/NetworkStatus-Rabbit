## 3. 登录方式

当前这组测试机器统一按 SSH 密钥登录。

已知信息：

1. 密钥目录在 D:/wingsrabbit-key/
2. 当前已经实机确认：四台机器都可以使用 D:/wingsrabbit-key/id_rsa_private 登录
3. 当前已经实机确认：四台机器登录用户都是 `root`

如果后续某台测试机改了用户名、端口或专用私钥，需要优先更新本文件。

## 4. 推荐用途

### 4.1 服务端 123.253.226.10

适合做的事情：

1. 验证 Docker Compose 部署
2. 验证后端 API、Web 页面、WebSocket、InfluxDB 数据链路
3. 验证节点管理、任务管理、告警、系统设置等主流程
4. 验证更新脚本、部署脚本和版本升级流程

### 4.2 客户端 123.253.226.11 到 123.253.226.13

适合做的事情：

1. 安装 Agent，验证注册、认证、心跳、任务同步
2. 做 ICMP、TCP、UDP、HTTP、DNS、MTR 协议联调
3. 做多节点互测，验证 internal 任务链路
4. 做掉线补传、节点上下线、告警恢复等行为测试
5. 验证 listen-port 开启后的内部 TCP/UDP 目标能力

## 5. 当前建议的测试组合

推荐按下面方式使用：

1. 123.253.226.10 作为 Center 服务端
2. 123.253.226.11 作为基础 Agent 节点
3. 123.253.226.12 作为第二个 Agent 节点，用于 internal 任务互测
4. 123.253.226.13 作为第三个 Agent 节点，用于并发、多协议、MTR 或异常场景补充测试

## 6. 新人接手时最短结论

如果只是想快速开始测试，先记住这几件事：

1. 服务端看 123.253.226.10
2. 客户端看 123.253.226.11 到 123.253.226.13
